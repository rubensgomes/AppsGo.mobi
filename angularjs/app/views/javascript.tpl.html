Is JavaScript a pass-by-reference or pass-by-value language?

The answer depends on the type of variable being passed.

Primitive typed variables (e.g. Boolean, Number, String) are stored in the stack. Therefore, when passing these types of variables to functions a copy of that variable value is made into the function argument which has its own stack. In this case the answer is passed-by-value.

Object and array typed variables are stored in the heap. Therefore, when passing these types of variables to functions a copy of the the object or array memory address in the heap is made into the function argument. In this case the answer is passed-by-reference.
